Plus de 1 000 Environnements Cloud Infectés par une Attaque sur le Scanner Trivy

Un groupe de pirates a réussi à compromettre Trivy, un scanner de vulnérabilités open source largement utilisé dans les pipelines de développement. Cette attaque a conduit à l'infection de plus de 1 000 environnements SaaS par un malware qui vole des clés API, des identifiants cloud et des tokens GitHub.

Trivy, un outil open source maintenu par Aqua Security, est conçu pour détecter des failles, des mauvaises configurations et des secrets exposés dans du code. Intégré dans les chaînes de déploiement continu (CI/CD) de nombreuses entreprises, Trivy est devenu une cible de choix pour les cybercriminels. Le groupe TeamTNT a exploité une mauvaise configuration dans le composant GitHub Action de la version 0.69.4 de Trivy, transformant ainsi un outil de sécurité en vecteur d'attaque.

Cette attaque met en lumière les risques associés aux outils open source et souligne l'importance de la vigilance constante dans la gestion des configurations et des mises à jour. Les entreprises doivent redoubler d'efforts pour sécuriser leurs environnements cloud et protéger leurs données sensibles.

Intel Booste les Performances de ses GPU Arc sous Linux

Intel vient de donner un coup de pouce significatif aux performances de ses GPU Arc sous Linux. Le pilote Vulkan open source d'Intel pour Linux a reçu une optimisation majeure qui améliore les performances des jeux DirectX 12 fonctionnant via Proton. Cette mise à jour, intégrée à Mesa 26.1, concerne les cartes graphiques Arc Alchemist et Battlemage.

Proposée pour la première fois en 2020, cette modification aura mis plus de cinq ans à voir le jour. L'optimisation porte sur la gestion du cache d'état graphique par le pilote ANV. En utilisant une combinaison de deux identifiants internes (Binding Table Pointer et Binding Table Index) au lieu d'un seul pour référencer les textures, le pilote peut éliminer certaines étapes de synchronisation qui ralentissaient le rendu.

Cette amélioration est une excellente nouvelle pour les joueurs sous Linux, qui bénéficieront désormais de performances accrues et d'une expérience de jeu plus fluide. Intel continue ainsi de renforcer son engagement envers la communauté open source et les utilisateurs de Linux.

MS-DOS Tourne Maintenant sur un Apple IIe, et C'est un Projet Open Source

Un développeur a réalisé un exploit impressionnant : faire tourner MS-DOS 2.0 sur l'Apple IIe, l'ordinateur personnel d'Apple sorti en 1983. Baptisé "reboot-camp-83", ce projet repose sur une carte d'extension équipée d'un processeur Intel 8088 à 8 MHz. Cette carte communique avec le processeur 6502 de l'Apple II, permettant ainsi l'exécution de MS-DOS sur cette machine vintage.

Seth Kushniryk, le cerveau derrière ce projet open source, a rendu le code accessible à tous. Pour faire fonctionner MS-DOS sur l'Apple IIe, il utilise une carte d'extension AD8088, fabriquée par ALF Products. Quarante ans après la sortie de l'Apple IIe, ce projet montre que l'innovation et la passion pour la technologie rétro sont toujours bien vivantes.

Ce projet est non seulement un hommage à l'histoire de l'informatique, mais aussi une démonstration brillante de ce que la communauté open source peut accomplir. Les amateurs de rétro-informatique et les développeurs curieux peuvent maintenant explorer et contribuer à ce projet fascinant.
