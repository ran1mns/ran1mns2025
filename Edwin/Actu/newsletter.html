Faites tourner Firefox dans votre terminal avec term.everything !

Imaginez pouvoir faire tourner Firefox, avec tous ses onglets, images et fonctionnalités, directement dans votre terminal. Et ce n'est pas tout : cela fonctionne même via SSH, depuis un serveur distant. Bienvenue dans le futur (ou peut-être le passé, qui sait ?) !

Term.everything est un compositeur Wayland construit from scratch en Go. Au lieu d'afficher l'image sur votre écran, il la convertit en caractères ANSI et l'affiche dans le terminal. Cela signifie que n'importe quelle application GUI Linux peut fonctionner dans votre terminal. Que ce soit Firefox, un gestionnaire de fichiers, un lecteur vidéo, ou même Doom (parce que si ça ne peut pas faire tourner Doom, ça ne compte pas vraiment). Le binaire est léger, ne pesant que quelques Mo, sous licence AGPL-3.0, et sans aucune dépendance externe.

Claude Code : Une fuite de code source qui coûte cher à Anthropic

Un incident majeur a frappé Anthropic : 60 Mo de source maps, ces fichiers permettant de remonter du code minifié à l'original, ont été accidentellement inclus dans un paquet npm. Résultat ? Le code source complet de Claude Code, un outil générant 2,5 milliards de dollars de revenus annuels, a été rendu public.

Que s'est-il passé exactement ? Hier, la version 2.1.88 du package `@anthropic-ai/claude-code` sur le registre npm contenait un fichier `.map` de 59,8 Mo, normalement réservé au débogage interne. Ce fichier contenait les pointeurs vers les 1 900 fichiers TypeScript originaux, en clair. Chaofan Shou, un développeur chez Solayer Labs, a repéré l'erreur et l'a partagée sur X. Avant qu'Anthropic ne puisse réagir, le code était déjà mirrored partout sur GitHub, avec plus de 41 500 forks en quelques heures. Autant dire que le dentifrice ne rentrera pas dans le tube !

Alerte sécurité : Axios, la bibliothèque JavaScript populaire, piratée pour installer un cheval de Troie

La bibliothèque JavaScript Axios, téléchargée plus de 100 millions de fois par semaine, a été compromise par un attaquant. Ce dernier a détourné le compte du mainteneur principal pour y injecter un malware multiplateforme ciblant macOS, Windows et Linux.

Tout a commencé avec le compte npm de jasonsaayman, le mainteneur principal d'Axios. L'attaquant a réussi à prendre le contrôle du compte, a changé l'adresse email vers un ProtonMail anonyme, et a publié deux versions malveillantes : axios 1.14.1 et axios 0.30.4. Cette attaque souligne l'importance de la sécurité dans les chaînes d'approvisionnement logiciel et la vigilance nécessaire pour protéger les comptes des mainteneurs de projets open source.

Mais un amendement déposé la semaine dernière par le sénateur Bernard Lotte (LR) propose d'étendre cette logique aux plus de 50 ans. Oui, vous avez bien lu ! L'amendement n°104-AP, intitulé "Protection des publics vulnérables face aux manipulations numériques", s'appuie sur les conclusions du comité d'évaluation. Le constat est sans appel : les seniors seraient les premiers relayeurs de fausses informations sur Facebook et WhatsApp, bien loin devant les ados.

Selon le rapport, 67% des partages de fake news sur Facebook en France viendraient des plus de 55 ans. Le reste se répartit entre X, Instagram et Telegram. Cette proposition soulève des questions sur la liberté d'expression et la responsabilité des utilisateurs sur les réseaux sociaux.
