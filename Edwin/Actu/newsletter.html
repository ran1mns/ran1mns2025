Pourquoi les navigateurs ralentissent-ils les minuteries JavaScript ? Décryptage par Nolan Lawson

Même si vous êtes un développeur JavaScript chevronné, vous pourriez être surpris d'apprendre que setTimeout(0) ne s'exécute pas instantanément. En réalité, cette minuterie peut prendre jusqu'à 4 millisecondes pour s'exécuter. Mais pourquoi ce délai ? Nolan Lawson nous éclaire sur ce mystère.

Les navigateurs modernes implémentent une technique appelée "clamping" pour les minuteries JavaScript. Cela signifie que même si vous définissez un délai de 0 milliseconde, le navigateur peut ajouter un délai minimal de 4 millisecondes. Cette pratique vise à optimiser les performances et à éviter une surcharge du processeur.

Par exemple, si vous exécutez le code suivant :

const start = performance.now();
setTimeout(() => {
// Likely 4ms
console.log(performance.now() - start);
});

Vous constaterez que le délai réel est souvent de 4 millisecondes, même si vous avez spécifié 0. Cette optimisation permet aux navigateurs de gérer plus efficacement les ressources et d'améliorer l'expérience utilisateur.

Cybersécurité

La sécurité quantique : l'avenir de la protection des données

La cybersécurité évolue vers de nouvelles dimensions avec l'avènement de la sécurité quantique. En 2026, les recherches sur les communications quantiques et les ordinateurs quantiques progressent rapidement, offrant des solutions potentielles pour protéger les données contre les attaques les plus sophistiquées. Les clés de chiffrement quantique, basées sur les principes de la mécanique quantique, promettent une sécurité inégalée, rendant les données pratiquement inviolables. Les gouvernements et les entreprises technologiques investissent dans cette technologie révolutionnaire, préparant le terrain pour une ère où la cybersécurité atteindra des niveaux de protection sans précédent.
