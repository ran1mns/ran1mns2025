Titre du Cours : L'Anglais Technique pour les Développeurs

Introduction

L'anglais est la langue universelle de l'informatique. Maîtriser un vocabulaire technique de base est essentiel pour lire de la documentation, comprendre des tutoriels, collaborer sur des projets open-source et trouver des solutions sur des forums comme Stack Overflow. Ce cours vous donne les clés pour vous sentir plus à l'aise avec l'anglais technique.

1. Vocabulaire Essentiel du Développement

Programmation (Software)

  • Variable: a container for storing data.
  • Function / Method: a block of code that performs a specific task.
  • Loop: a control structure that repeats a block of code (for, while).
  • Conditional / Statement: a control structure that executes code based on a condition (if, else).
  • Bug: an error in a program that causes it to produce an incorrect result.
  • Debugging: the process of finding and fixing bugs.
  • Framework / Library: a collection of pre-written code to help develop applications faster.

Gestion de Projet et Versioning (Git)

  • Repository (Repo): a directory where your project's files are stored.
  • Commit: a snapshot of your files at a specific time.
  • Branch: an independent line of development.
  • Merge: to combine changes from different branches.
  • Pull Request (PR): a request to merge your changes into another branch.
  • Issue: a bug report or a feature request.

2. Comprendre la Documentation Technique

La documentation en anglais suit souvent une structure similaire. Savoir la reconnaître vous fera gagner du temps.

  • Getting Started: Le guide de démarrage rapide.
  • Prerequisites: Les prérequis (logiciels à installer avant de commencer).
  • Installation: Les étapes pour installer l'outil.
  • Usage / API Reference: Comment utiliser l'outil ou la librairie, avec la description de chaque fonction.
  • Troubleshooting / FAQ: Guide de dépannage et foire aux questions.

3. Raconter ses Actions : le Prétérit (Simple Past)

Le prétérit est essentiel pour décrire ce que vous avez fait sur un projet, par exemple dans un rapport ou lors d'un stand-up meeting.

  • Verbes réguliers : on ajoute "-ed". (I fixed the bug.)
  • Verbes irréguliers : il faut les apprendre ! (I wrote the documentation.)

// Exemple de rapport journalier
- Yesterday, I implemented the user authentication feature.
- I wrote unit tests for the login function.
- Then, I pushed my changes to the `develop` branch.

4. Prépositions Utiles

  • in: Utilisé pour les langages, les fichiers, les environnements.
    • "I wrote the script in Python."
    • "The configuration is in the `.env` file."
  • on: Utilisé pour les plateformes, les serveurs, les branches.
    • "I'm working on a new feature."
    • "The code is running on the production server."
    • "I created a new branch on our repository."
  • at: Utilisé pour des points précis.
    • "The error occurs at line 52."

5. Phrases Courantes (Stack Overflow, GitHub...)

Voici quelques phrases que vous rencontrerez très souvent :

  • "I'm trying to..." (J'essaie de...)
  • "I'm getting an error..." (Je reçois une erreur...)
  • "How can I achieve this?" (Comment puis-je réaliser cela ?)
  • "This is a known issue." (C'est un problème connu.)
  • "Could you provide a code snippet?" (Pourriez-vous fournir un extrait de code ?)
  • "This pull request fixes a bug where..." (Cette pull request corrige un bug où...)

Exercices Pratiques

  1. Traduisez ce message de commit : "feat(auth): add login functionality and user validation".
  2. Décrivez cette fonction en une phrase en anglais : 
    
def calculate_sum(numbers):
  total = 0
  for number in numbers:
    total += number
  return total
  3. Trouvez la signification de l'acronyme "DRY" dans le contexte du développement logiciel.
Corrigés
  1. Traduction : "Nouvelle fonctionnalité (authentification) : ajout de la fonctionnalité de connexion et de la validation de l'utilisateur".
  2. Description : "This function takes a list of numbers and returns their sum." (Cette fonction prend une liste de nombres et retourne leur somme.)
  3. DRY : "Don't Repeat Yourself" (Ne vous répétez pas). C'est un principe de développement qui vise à réduire la duplication de code.
