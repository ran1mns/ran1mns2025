Titre du Cours : Interconnexion des Ordinateurs et Réseaux

Introduction

L'interconnexion des ordinateurs est essentielle pour permettre la communication et le partage de données à travers le monde. Ce cours vous guidera à travers les concepts fondamentaux des réseaux informatiques, des types de réseaux aux outils et protocoles utilisés pour leur gestion.

Concepts Clés

Réseaux de Type Maillé

Un réseau maillé est un type de réseau où chaque nœud est connecté à plusieurs autres nœuds. Ce type de réseau est utilisé pour éviter les pannes, car il y a toujours une liaison secondaire disponible.


Exemple : Les câbles sous-marins et terrestres qui relient différentes parties du monde.

LAN (Local Area Network) et WAN (Wide Area Network)

Un LAN est un réseau local qui couvre une petite zone géographique, comme un bureau ou une maison. Un WAN, en revanche, couvre une grande zone géographique, comme un pays ou même le monde entier.


Exemple : Un LAN peut être le réseau de votre maison, tandis qu'un WAN peut être le réseau de votre entreprise avec des bureaux dans différentes villes.

Émulation et Simulation

L'émulation permet de créer des machines virtuelles pour tester des environnements réseau. La simulation utilise des modèles mathématiques pour prévoir le comportement des réseaux.


Exemple : Utiliser Cisco Packet Tracer pour simuler un réseau avec des PC, des switchs, des routeurs et des serveurs.

Switch, Routeur et Firewall

Un switch est utilisé pour interconnecter des dispositifs dans un LAN. Un routeur est utilisé pour interconnecter des LAN et des WAN. Un firewall protège le réseau contre les accès non autorisés.


Exemple : Un switch peut être utilisé pour connecter plusieurs ordinateurs dans un bureau, tandis qu'un routeur peut connecter ce bureau à Internet.

Types de Câbles Ethernet

Les câbles Ethernet, Fast Ethernet et Giga Ethernet offrent des débits de plus en plus élevés.


Exemple : Un câble Ethernet standard offre un débit de 10 Mbps, tandis qu'un câble Giga Ethernet offre un débit de 1 Gbps.

Adresse Broadcast

L'adresse broadcast est la dernière adresse d'un réseau et est utilisée pour envoyer un message à tous les dispositifs du réseau.


Exemple : Dans le réseau 192.168.1.0, l'adresse broadcast est 192.168.1.255.

VLSM (Variable Length Subnet Masking)

Le VLSM permet de diviser un réseau en sous-réseaux de différentes tailles pour une utilisation plus efficace des adresses IP.


Exemple : Utiliser VLSM pour créer des sous-réseaux avec des masques de sous-réseau différents pour des départements de tailles différentes.

Schémas IP Binaire/Décimal et Décimal Pointé

Les adresses IP peuvent être représentées en binaire, décimal ou décimal pointé. Il est important de comprendre ces représentations pour configurer et gérer les réseaux.


Exemple : L'adresse IP 192.168.1.1 en binaire est 11000000.10101000.00000001.00000001.

Réseau NAT/PAT

Le NAT (Network Address Translation) permet de traduire les adresses IP privées en adresses IP publiques. Le PAT (Port Address Translation) est une extension du NAT qui utilise des ports pour distinguer les connexions.


Exemple : Utiliser NAT pour permettre à plusieurs ordinateurs dans un réseau privé d'accéder à Internet avec une seule adresse IP publique.

Commande ipconfig /all

La commande ipconfig /all fournit des informations détaillées sur la configuration réseau d'un ordinateur.


Exemple : Utiliser ipconfig /all pour vérifier l'adresse IP, le masque de sous-réseau, la passerelle et les serveurs DNS.

Passerelle et Serveurs

La passerelle est la porte de sortie du réseau, généralement l'avant-dernière adresse de la plage. Les serveurs dans le même réseau ont souvent des adresses se terminant par .101, .102, etc.


Exemple : Dans le réseau 192.168.1.0, la passerelle peut être 192.168.1.254.

DHCP (Dynamic Host Configuration Protocol)

Le DHCP est un protocole qui attribue automatiquement des adresses IP aux dispositifs d'un réseau à partir d'un pool d'adresses.


Exemple : Configurer un serveur DHCP pour attribuer des adresses IP aux ordinateurs d'un réseau local.

Exercices Pratiques

  1. Configurez un réseau local avec deux ordinateurs, un switch et un routeur. Assurez-vous que les deux ordinateurs peuvent communiquer entre eux.
  2. Utilisez la commande ipconfig /all pour obtenir des informations détaillées sur la configuration réseau de votre ordinateur. Identifiez l'adresse IP, le masque de sous-réseau, la passerelle et les serveurs DNS.
  3. Configurez un serveur DHCP pour attribuer des adresses IP à cinq ordinateurs dans un réseau local. Vérifiez que chaque ordinateur reçoit une adresse IP unique.
Corrigé 

1. Configuration du réseau local :
   - Connectez les deux ordinateurs au switch.
   - Connectez le switch au routeur.
   - Configurez les adresses IP des ordinateurs et du routeur pour qu'ils soient dans le même sous-réseau.
   - Testez la connectivité en utilisant la commande ping.

2. Utilisation de ipconfig /all :
   - Ouvrez l'invite de commande.
   - Tapez ipconfig /all et appuyez sur Entrée.
   - Notez les informations affichées pour l'adresse IP, le masque de sous-réseau, la passerelle et les serveurs DNS.

3. Configuration du serveur DHCP :
   - Installez et configurez un serveur DHCP sur un ordinateur.
   - Définissez une plage d'adresses IP à attribuer.
   - Connectez les cinq ordinateurs au réseau.
   - Vérifiez que chaque ordinateur reçoit une adresse IP unique en utilisant la commande ipconfig.
