Titre du Cours : Déploiement d'Applications sur un Serveur Linux

Introduction

Ce cours vous guidera à travers les étapes fondamentales du déploiement d'une application web sur un serveur Linux. Nous verrons comment préparer le serveur, transférer vos fichiers, configurer un serveur web et une base de données pour rendre votre application accessible en ligne.

1. Préparation du Serveur

Systèmes d'Exploitation pour Serveurs

La grande majorité des serveurs web tournent sous des distributions Linux (comme Debian, Ubuntu Server, CentOS) pour leur stabilité, leur sécurité et leur gratuité. Ces serveurs sont généralement gérés via la ligne de commande, sans interface graphique.

Connexion à distance avec SSH

Pour vous connecter à votre serveur, vous utiliserez le protocole SSH (Secure Shell), qui assure une communication chiffrée.


# Syntaxe de connexion SSH
ssh utilisateur@adresse_ip_du_serveur

Mises à jour et Commandes de Base

Une fois connecté, la première étape est de mettre à jour le système.


# Mettre à jour la liste des paquets et les paquets installés
sudo apt update && sudo apt upgrade

Quelques commandes Linux essentielles : ls (lister les fichiers), cd (changer de répertoire), mkdir (créer un répertoire), cat (afficher un fichier), nano (éditer un fichier).

2. Déploiement de l'Application

Pour déployer votre application, vous devez transférer ses fichiers sur le serveur. Vous pouvez utiliser des outils comme scp (Secure Copy) ou git.


# Exemple avec scp pour copier un dossier local vers le serveur
scp -r /chemin/local/mon-projet utilisateur@adresse_ip_du_serveur:/var/www/

3. Configuration du Serveur Web

Un serveur web est un logiciel qui reçoit les requêtes HTTP des clients (navigateurs) et leur envoie les pages web correspondantes. Les plus populaires sont Apache et Nginx.

Exemple avec Apache


# Installer Apache
sudo apt install apache2

# Le dossier par défaut pour les sites web est /var/www/html
# Placez vos fichiers dans ce dossier ou configurez un "Virtual Host"

4. Configuration de la Base de Données

Si votre application nécessite une base de données, vous devrez l'installer et la configurer. MySQL et PostgreSQL sont des choix courants.

Exemple avec MySQL


# Installer le serveur MySQL
sudo apt install mysql-server

# Lancer le script de sécurisation
sudo mysql_secure_installation

# Se connecter et créer une base de données et un utilisateur
sudo mysql
CREATE DATABASE ma_base;
CREATE USER 'mon_utilisateur'@'localhost' IDENTIFIED BY 'mot_de_passe_solide';
GRANT ALL PRIVILEGES ON ma_base.* TO 'mon_utilisateur'@'localhost';
FLUSH PRIVILEGES;
EXIT;

5. Sauvegarde des Données

La sauvegarde est cruciale. La règle du 3-2-1 est un excellent principe de base :

  • 3 copies de vos données.
  • Sur 2 supports différents (ex: disque dur du serveur + disque externe).
  • Dont 1 copie hors site (ex: dans le cloud).

Exercices Pratiques

  1. Prise en main d'un serveur : Si vous avez accès à un VPS ou une machine virtuelle Linux, connectez-vous en SSH et mettez à jour le système.
  2. Installation d'un serveur web : Installez Apache ou Nginx sur votre serveur. Vérifiez qu'il fonctionne en accédant à l'adresse IP du serveur dans votre navigateur.
  3. Déploiement d'un site statique : Créez une simple page index.html et transférez-la sur votre serveur dans le bon répertoire (/var/www/html pour Apache) pour la rendre accessible en ligne.
  4. Installation de MySQL : Installez et sécurisez MySQL sur votre serveur.
