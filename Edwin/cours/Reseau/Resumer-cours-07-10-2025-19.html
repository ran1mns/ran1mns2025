Titre du Cours : Déploiement d'Applications sur Serveurs

Introduction

Dans ce cours, nous allons explorer comment déployer nos applications sur des serveurs. Nous aborderons les concepts essentiels tels que l'hébergement de bases de données, le fonctionnement des réseaux, et les systèmes d'exploitation des serveurs. Ce cours est crucial pour comprendre comment rendre vos applications accessibles en ligne de manière efficace et sécurisée.

Concepts Clés

Héberger la Base de Données

L'hébergement de la base de données est une étape essentielle pour assurer que vos applications puissent stocker et récupérer des données de manière fiable. Les bases de données peuvent être hébergées sur le même serveur que l'application ou sur un serveur dédié. Voici un exemple de configuration de base de données MySQL sur un serveur Linux:


# Installer MySQL
sudo apt-get update
sudo apt-get install mysql-server

# Sécuriser l'installation MySQL
sudo mysql_secure_installation

# Se connecter à MySQL
mysql -u root -p

Fonctionnement d'un Réseau

Pour déployer une application sur un serveur, il est crucial de comprendre comment fonctionne un réseau. Les réseaux permettent aux ordinateurs de communiquer entre eux. Voici un schéma simplifié du fonctionnement d'un réseau:


Hardware -> OS -> Application

L'application interagit avec le système d'exploitation (OS), et cette interaction peut varier en fonction de l'OS utilisé.

Systèmes d'Exploitation des Serveurs

La majorité des serveurs utilisent Linux comme système d'exploitation. Les hébergeurs ne fournissent généralement pas d'interfaces graphiques sur leurs serveurs, ce qui signifie que vous devrez utiliser des lignes de commande pour interagir avec le serveur. Voici quelques commandes de base pour naviguer dans un système Linux:


# Lister les fichiers et dossiers
ls

# Changer de répertoire
cd /path/to/directory

# Afficher le contenu d'un fichier
cat filename

Historique et Évolution des Systèmes Informatiques

Avant 1981, l'informatique était centralisée. Les machines étaient souvent fabriquées par le même constructeur et utilisaient des systèmes multi-utilisateurs. En 1981, IBM a lancé le premier PC, et Bill Gates a proposé MS DOS avec le concept de licence. Les ordinateurs ont commencé à être interconnectés, d'abord par disquettes, puis par câbles Ethernet.

Protocoles de Communication

Les protocoles de communication sont des règles qui permettent aux ordinateurs de communiquer entre eux sur un réseau. Parmi les protocoles les plus courants, on trouve HTTP, TCP/IP, SMTP, FTP, et DHCP. Voici un exemple de configuration d'un serveur web utilisant HTTP:


# Installer Apache
sudo apt-get update
sudo apt-get install apache2

# Démarrer Apache
sudo systemctl start apache2

# Vérifier le statut d'Apache
sudo systemctl status apache2

Réseaux Locaux (LAN)

Pour relier plusieurs ordinateurs, on utilise un switch. Un réseau local (LAN) permet de connecter des ordinateurs sans accès à Internet. Voici un schéma simplifié d'un réseau local:


Computer 1 -> Switch -> Computer 2

Règles de Sauvegarde

Il est crucial de suivre des règles de sauvegarde pour protéger vos données. La règle des 3-2-1 est souvent recommandée:

  • 3 versions du fichier
  • Sur 2 supports différents
  • 1 copie en interne et 1 copie ailleurs
