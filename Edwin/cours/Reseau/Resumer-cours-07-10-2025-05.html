Titre du Cours : Réseaux et Hébergement Web

Introduction

Ce cours va vous rafraîchir la mémoire sur les adresses IP, le DHCP, les classes d'adresses IP, et les différents types d'hébergement web. Vous comprendrez comment les adresses IP sont allouées, comment le DNS fonctionne, et quelles sont les options d'hébergement disponibles pour vos sites web.

Concepts Clés

DHCP (Dynamic Host Configuration Protocol)

Le DHCP est un service qui alloue automatiquement des adresses IP aux appareils sur un réseau. Il est souvent installé sur le routeur, mais peut aussi être installé sur une machine, parfois virtuelle. C'est un programme qui tourne en permanence sur le réseau.

Adresses IP

Une adresse IP est constituée de 4 éléments pour IPv4 et de 8 éléments pour IPv6. Par exemple, une adresse IPv4 ressemble à ceci :


192.168.1.1

Avec l'augmentation du nombre d'équipements, IPv6 a été créé pour fournir plus d'adresses IP. Une adresse IPv6 ressemble à ceci :


2001:0db8:85a3:0000:0000:8a2e:0370:7334

Classes d'Adresses IP

Les adresses IPv4 sont divisées en classes :

  • Classe A : 0 à 126 (numéro de réseau sur la première position, numéro d'hôte sur les trois positions restantes)
  • Classe B : 128 à 191 (numéro de réseau sur les deux premières positions, numéro d'hôte sur les deux positions restantes)
  • Classe C : 192 à 223 (numéro de réseau sur les trois premières positions, numéro d'hôte sur la position restante)

Les classes D et E existent aussi, mais elles ne sont pas accessibles au grand public.

Découpage en Plages

Les adresses IP sont découpées en plages allouées par les fournisseurs d'accès. Par exemple, les adresses commençant par 10, 172, ou 192 sont réservées pour les réseaux locaux et n'existent pas sur Internet.

DNS (Domain Name System)

Le DNS fait le lien entre le nom de domaine et l'adresse IP où le site est stocké. Par exemple, lorsque vous tapez "www.example.com" dans votre navigateur, le DNS traduit ce nom en une adresse IP.

Types d'Hébergement

Il existe plusieurs types d'hébergement web :

  • Hébergement mutualisé : plusieurs sites web partagent le même serveur.
  • VPS (Virtual Private Server) : un serveur virtuel privé, souvent avec une distribution Linux comme Debian.
  • Location d'un serveur physique entier : un serveur dédié uniquement à votre site web.

Exercices Pratiques

  1. Recherchez et expliquez la différence entre une VM (Virtual Machine) et un container.
  2. Utilisez un outil comme "whois" pour trouver le propriétaire d'un nom de domaine de votre choix.
  3. Configurez un serveur DHCP sur une machine virtuelle et allouez une adresse IP à un autre appareil sur le même réseau.
Corrigé 


VM (Virtual Machine) : Une machine virtuelle est une émulation d'un ordinateur physique. Elle utilise un hyperviseur pour gérer les ressources matérielles.

Container : Un container est une méthode de virtualisation au niveau du système d'exploitation. Il partage le noyau de l'OS avec d'autres containers, ce qui le rend plus léger et plus rapide qu'une VM.
