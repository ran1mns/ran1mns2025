Titre du Cours : Fondamentaux des Réseaux et de l'Hébergement Web

Introduction

Ce cours vous propose une révision des concepts essentiels des réseaux informatiques et de l'hébergement web. Nous explorerons le fonctionnement des adresses IP, l'allocation dynamique avec DHCP, la résolution de noms avec DNS, et les différentes options pour héberger un site web.

Le Modèle OSI

Le modèle OSI (Open Systems Interconnection) est un modèle conceptuel qui divise les communications réseau en sept couches. Chaque couche a un rôle spécifique, ce qui permet de standardiser les protocoles.

  1. Physique : Transmission des bits sur le média (câbles, ondes).
  2. Liaison de données : Adressage MAC, détection d'erreurs.
  3. Réseau : Routage des paquets, adressage IP.
  4. Transport : Connexion de bout en bout (TCP, UDP).
  5. Session : Gestion des sessions de communication.
  6. Présentation : Formatage des données, chiffrement.
  7. Application : Protocoles de haut niveau (HTTP, FTP, SMTP).

Adressage IP

Une adresse IP (Internet Protocol) est un numéro d'identification unique attribué à chaque appareil connecté à un réseau.

IPv4 vs IPv6

IPv4 : Codée sur 32 bits (4 octets), elle se présente sous la forme 192.168.1.1. L'épuisement des adresses IPv4 a conduit à la création d'IPv6.

IPv6 : Codée sur 128 bits, elle se présente sous la forme 2001:0db8:85a3:0000:0000:8a2e:0370:7334 et offre un espace d'adressage quasi illimité.

Masque de Sous-Réseau

Le masque de sous-réseau (ex: 255.255.255.0) permet de distinguer la partie réseau de la partie hôte dans une adresse IP. Il définit la taille du réseau local.

DHCP (Dynamic Host Configuration Protocol)

Le DHCP est un protocole qui permet à un serveur d'attribuer automatiquement une adresse IP et d'autres paramètres réseau (masque, passerelle, DNS) aux appareils d'un réseau. Cela simplifie grandement l'administration.

DNS (Domain Name System)

Le DNS est un service qui traduit les noms de domaine (ex: www.google.com), faciles à retenir pour les humains, en adresses IP (ex: 142.250.179.196), compréhensibles par les machines.

Hébergement Web

L'hébergement web consiste à stocker les fichiers d'un site web sur un serveur pour le rendre accessible sur Internet.

  • Hébergement Mutualisé : Économique, mais les ressources sont partagées avec d'autres sites. Idéal pour les petits sites personnels.
  • VPS (Virtual Private Server) : Un serveur virtuel qui offre plus de contrôle et de ressources dédiées qu'un hébergement mutualisé. Bon compromis performance/prix.
  • Serveur Dédié : Un serveur physique entièrement à votre disposition. Offre des performances et une sécurité maximales, mais est plus cher et complexe à gérer.
  • Cloud Hosting : Hébergement sur un réseau de serveurs connectés. Très flexible et scalable, vous ne payez que ce que vous consommez.

Exercices Pratiques

  1. Identifier votre configuration IP : Ouvrez un terminal (CMD sur Windows, Terminal sur macOS/Linux) et tapez ipconfig (Windows) ou ifconfig (macOS/Linux). Identifiez votre adresse IP, votre masque de sous-réseau et votre passerelle par défaut.
  2. Interroger le DNS : Utilisez la commande nslookup google.com dans votre terminal pour trouver l'adresse IP associée au nom de domaine de Google.
  3. Comparer les offres d'hébergement : Choisissez deux hébergeurs web connus (ex: OVH, Hostinger, SiteGround) et comparez leurs offres d'hébergement mutualisé et VPS en termes de prix, de ressources (CPU, RAM, stockage) et de fonctionnalités.
