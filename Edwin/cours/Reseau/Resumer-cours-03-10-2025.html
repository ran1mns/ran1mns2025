Architecture Réseau & Virtualisation

Hébergement & Serveurs

Le choix de l'infrastructure d'hébergement dépend des besoins en ressources et en contrôle.

  • Mutualisé : Plusieurs sites partagent les mêmes ressources. Économique mais limité.
  • VPS (Virtual Private Server) : Une partition virtuelle d'un serveur physique avec ses propres ressources dédiées.
  • Serveur Dédié : Un serveur physique entier réservé à un seul client. Puissance maximale.
  • Cloud : Infrastructure scalable et distribuée (AWS, Azure, GCP).

La Virtualisation

La virtualisation permet de faire tourner plusieurs systèmes d'exploitation sur une seule machine physique.

Types d'Hyperviseurs :

  • Type 1 (Bare Metal) : S'installe directement sur le matériel (ex: ESXi, Proxmox).
  • Type 2 (Hosted) : S'exécute sur un système d'exploitation hôte (ex: VirtualBox, VMware Workstation).

Contrairement aux machines virtuelles, les conteneurs (Docker) partagent le noyau de l'OS hôte, ce qui les rend beaucoup plus légers.

Modèles OSI & TCP/IP

La communication réseau repose sur des couches normalisées.

Protocoles & Ports Communs :

Service Protocole Port
Web (Non-sécurisé) HTTP 80
Web (Sécurisé) HTTPS 443
Accès à distance SSH 22
Transfert de fichiers FTP 21
Base de données MySQL 3306

Sécurité & SSH

Le protocole SSH (Secure Shell) est indispensable pour l'administration sécurisée à distance.

Authentification par clés :

Plus sécurisée que les mots de passe. Repose sur une paire de clés :

  • Clé Publique : Déposée sur le serveur.
  • Clé Privée : Conservée précieusement par l'utilisateur.
ssh-keygen -t rsa -b 4096
ssh-copy-id utilisateur@serveur
