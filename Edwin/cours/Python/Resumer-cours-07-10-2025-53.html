Concept de File d'Attente (FIFO)

La file d'attente est une structure de données où le premier élément inséré est le premier à être retiré (First In, First Out). C'est un concept fondamental en informatique.


# Exemple de file d'attente en Python
from collections import deque
file_attente = deque(['a', 'b', 'c'])
while file_attente:
    print(file_attente.popleft())

Représentation Visuelle de l'Algorithme

Pour représenter visuellement un algorithme, on utilise des logigrammes (flowcharts). Un logigramme est un schéma conventionné pour représenter les étapes d'un algorithme.


# Exemple de pseudo-code pour "Devine le nombre"
ALGORITHM GuessTheNumber

VARIABLES
    targetNumber: INT  // Nombre à faire deviner
    attempt: INT
CONSTANTS
    MIN: INT <-- 1
    MAX: INT <-- 1000  // Éviter les "magic numbers"

BEGIN
    targetNumber <-- RANDOM(MIN, MAX)

    DO
        READ(attempt)
        IF attempt > targetNumber
            PRINT("Trop grand")
        ELSE IF attempt < targetNumber
            PRINT("Trop petit")
        ELSE
            PRINT("Gagné")
        EN-IF
    WHILE(attempt != targetNumber)

END

Logigramme

Un logigramme pour le même algorithme peut être représenté comme suit :

  1. Forme ronde avec "START" (point d'entrée)
  2. Forme rectangle "targetNumber=RAND(1,1000)" (étape informatique)
  3. Forme parallélépipède "READ(attempt)" (action de l'utilisateur)
  4. Forme lozange "attempt > targetNumber" (condition)
  5. Forme parallélo "trop grand" (première sortie)
  6. Forme lozange "attempt < targetNumber" (condition)
  7. Forme parallélo "trop petit" (si vrai)
  8. Forme parallélo "PRINT('gagné')" (si vrai)
  9. Forme ronde "END" (point de sortie)

Une flèche de "trop petit" et "trop grand" vers la liaison entre target et attempt matérialise le while.
