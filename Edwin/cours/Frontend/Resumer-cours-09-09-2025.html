Fondamentaux du Web & Accessibilité

Accessibilité Numérique (A11y)

Concevoir pour le web, c'est concevoir pour TOUS. L'accessibilité n'est pas une option, c'est une responsabilité légale et éthique.

RGAA & WCAG :

Le RGAA (Référentiel Général d’Amélioration de l’Accessibilité) est le cadre français basé sur les standards internationaux WCAG.

  • Perceptible : Information et composants d'interface doivent être perceptibles.
  • Utilisable : Les composants doivent être utilisables (clavier, voix).
  • Compréhensible : L'information doit être claire.
  • Robuste : Le contenu doit être compatible avec les technologies d'assistance.

Structure Sémantique HTML5

Utiliser les bonnes balises pour le bon contenu améliore le SEO et l'accessibilité.


<header> <!-- En-tête du site -->
  <nav> <!-- Menu de navigation --> </nav>
</header>

<main> <!-- Contenu principal -->
  <section> <!-- Groupe thématique -->
    <article> <!-- Unité de contenu autonome --> </article>
  </section>
</main>

<footer> <!-- Pied de page --> </footer>

Hiérarchie des Titres (H1-H6) :

Ne sautez jamais de niveau ! La structure doit être logique : H1 > H2 > H3.

Le rôle du CSS

Le CSS (Cascading Style Sheets) gère la présentation. Il doit être séparé du contenu (HTML).

Bonnes pratiques :
  • Ne pas utiliser la couleur comme seul moyen d'information.
  • Assurer un contraste suffisant entre le texte et le fond.
  • Utiliser des unités relatives (rem, em) plutôt que des pixels fixes.

Introduction à la Confidentialité (RGPD)

Le Règlement Général sur la Protection des Données impose des règles strictes sur la collecte et le traitement des données personnelles.

  • Transparence : Informer l'utilisateur.
  • Consentement : Obtenir l'accord explicite (Cookies).
  • Droit à l'oubli : Permettre la suppression des données.
