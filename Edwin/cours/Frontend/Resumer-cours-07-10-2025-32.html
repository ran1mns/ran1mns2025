Titre du Cours : Introduction à Git et GitHub

Introduction

Git est un moteur de gestion de versions, tandis que GitHub est une plateforme qui exploite ce moteur. Git permet de sauvegarder chaque étape de votre projet, de travailler à plusieurs et de revenir en arrière si besoin. GitHub, quant à lui, facilite la collaboration et l'intégration continue (CI/CD).

Concepts Clés

Installation de Git

Pour commencer, vous devez installer Git sur votre machine. Vous pouvez télécharger l'installateur depuis le site officiel de Git et suivre les instructions.


      # Exemple de commande pour vérifier l'installation de Git
      git --version

Repository

Un repository (ou dépôt) est l'équivalent d'un projet. C'est là que vous stockez tous vos fichiers et dossiers liés à votre projet.


      # Créer un nouveau repository
      git init mon-projet

Commit

Un commit est un point de sauvegarde. Il est conseillé de créer de petits commits et de les nommer clairement pour faciliter la gestion du projet.


      # Ajouter des fichiers à l'index
      git add .
      # Créer un commit
      git commit -m "Ajout de la fonctionnalité X"

Tag

Un tag permet de marquer un commit pour lui donner une version, ce qui est utile pour retrouver la version qui a été livrée.


      # Créer un tag
      git tag -a v1.0 -m "Version 1.0"

Branche

Une branche permet de créer une autre version du projet pour une nouvelle fonctionnalité. Cela permet à deux projets de cohabiter.


      # Créer une nouvelle branche
      git checkout -b nouvelle-fonctionnalite

Merge

Le merge permet de prendre les commits d'une branche et de les réinjecter dans la branche principale pour créer un nouveau commit.


      # Fusionner une branche dans la branche principale
      git checkout main
      git merge nouvelle-fonctionnalite

Rebase

Le rebase modifie la structure de l'arbre pour faire débuter la branche ailleurs. C'est une pratique risquée mais utile pour éviter les conflits lors du merge.


      # Rebaser une branche
      git checkout nouvelle-fonctionnalite
      git rebase main

Workflow

Un workflow définit comment une équipe utilise Git. Il existe plusieurs workflows comme GitFlow, GitHub Flow et Trunk Based Development.

Conventions de Commit

Un commit doit représenter une idée unique. Il est important de suivre des conventions pour rendre l'historique lisible.


      # Exemple de message de commit
      git commit -m "feat(auth): Ajout de la fonctionnalité d'authentification"

.gitignore

Le fichier .gitignore contient les fichiers que Git doit ignorer lors du push, comme les informations sensibles ou les fichiers cache.


      # Exemple de fichier .gitignore
      node_modules/
      .env

Vous pouvez utiliser gitignore.io pour générer automatiquement votre fichier .gitignore.

Exercices Pratiques

  1. Créez un nouveau repository local et faites votre premier commit.
  2. Créez une nouvelle branche pour ajouter une nouvelle fonctionnalité et fusionnez-la ensuite dans la branche principale.
  3. Ajoutez un fichier .gitignore à votre projet pour ignorer les fichiers sensibles.

        git checkout main
