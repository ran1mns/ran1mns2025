Titre du Cours : Introduction à la Ligne de Commande Linux

Introduction

La ligne de commande (ou terminal) est une interface texte puissante pour interagir avec un système Linux. La maîtriser est une compétence fondamentale pour tout développeur, car elle permet de gérer des serveurs, d'automatiser des tâches et de travailler plus efficacement.

1. Naviguer dans le Système de Fichiers

  • pwd (Print Working Directory) : Affiche le chemin complet du répertoire où vous vous trouvez.
  • ls (List) : Liste le contenu du répertoire courant.
    • ls -l : Affiche une liste détaillée (permissions, propriétaire, taille...).
    • ls -a : Affiche tous les fichiers, y compris les fichiers cachés (ceux qui commencent par un .).
    • ls -la : Combine les deux options.
  • cd (Change Directory) : Permet de se déplacer dans l'arborescence.
    • cd /chemin/vers/un/dossier : Se déplace vers le dossier spécifié.
    • cd .. : Remonte au répertoire parent.
    • cd ~ ou cd : Retourne au répertoire personnel de l'utilisateur.

2. Manipuler les Fichiers et Répertoires

Création

  • touch [nom_fichier] : Crée un fichier vide.
  • mkdir [nom_dossier] : Crée un nouveau répertoire.
    • mkdir -p projets/web/images : L'option -p crée les répertoires parents s'ils n'existent pas.

Copie, Déplacement et Renommage

  • cp [source] [destination] (Copy) : Copie un fichier ou un répertoire.
  • mv [source] [destination] (Move) : Déplace ou renomme un fichier ou un répertoire.

# Renommer un fichier
mv ancien_nom.txt nouveau_nom.txt

# Déplacer un fichier dans un autre dossier
mv fichier.txt /home/user/documents/

Suppression

  • rm [nom_fichier] (Remove) : Supprime un fichier. Attention, cette action est irréversible !
  • rmdir [nom_dossier] (Remove Directory) : Supprime un répertoire vide.
  • rm -r [nom_dossier] : Supprime un répertoire et tout son contenu (fichiers et sous-dossiers). À utiliser avec une extrême prudence.

3. Consulter et Éditer des Fichiers

  • cat [nom_fichier] : Affiche tout le contenu d'un fichier dans le terminal.
  • less [nom_fichier] : Affiche le contenu d'un fichier page par page (très utile pour les longs fichiers). Appuyez sur q pour quitter.
  • nano [nom_fichier] : Ouvre un éditeur de texte simple directement dans le terminal.

4. Gestion du Système (Super-utilisateur)

sudo

La commande sudo (Super User Do) permet d'exécuter une commande avec les privilèges d'administrateur (root). Elle est nécessaire pour les actions qui affectent le système, comme l'installation de logiciels.


# Mettre à jour la liste des paquets disponibles (Debian/Ubuntu)
sudo apt update

# Installer le programme 'tree'
sudo apt install tree

Exercice Pratique : Mettre en place un projet

Utilisez uniquement la ligne de commande pour réaliser les étapes suivantes dans votre répertoire personnel :

  1. Créez un répertoire mon_site_web.
  2. Déplacez-vous à l'intérieur de ce répertoire.
  3. Créez deux sous-répertoires : pages et css.
  4. Dans le répertoire racine mon_site_web, créez un fichier index.html.
  5. Dans le répertoire css, créez un fichier style.css.
  6. Affichez l'arborescence de votre projet (astuce : installez et utilisez la commande tree).
Corrigé 

mkdir mon_site_web
cd mon_site_web
mkdir pages css
touch index.html
touch css/style.css
sudo apt install tree # Si non installé
tree
