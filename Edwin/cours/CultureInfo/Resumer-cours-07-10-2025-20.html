Titre du Cours : Déploiement d'Applications sur des Serveurs

Introduction

Dans ce cours, nous allons explorer comment déployer des applications sur des serveurs. Nous aborderons les bases de l'hébergement de bases de données, le fonctionnement des réseaux, et les systèmes d'exploitation des serveurs. Ce cours est essentiel pour comprendre comment vos applications interagissent avec les serveurs et les réseaux.

Concepts Clés

Héberger la Base de Données

L'hébergement de la base de données est crucial pour le bon fonctionnement de votre application. Une base de données bien hébergée garantit la disponibilité et la sécurité des données. Les serveurs utilisent principalement des systèmes d'exploitation Linux, et il est important de comprendre comment interagir avec ces systèmes via la ligne de commande.

Fonctionnement d'un Réseau

Un réseau permet la communication entre différents ordinateurs. Avant 1981, l'informatique était centralisée, avec des machines spécifiques à chaque constructeur. En 1981, IBM a lancé le premier PC, et Bill Gates a introduit MS-DOS avec le concept de licence. Les ordinateurs ont commencé à être interconnectés via des câbles, et des protocoles de communication comme HTTP, TCP/IP, SMTP, FTP, et DHCP ont été développés pour standardiser cette communication.


      
      
Exemple de requête HTTP :

      
GET /index.html HTTP/1.1
Host: www.example.com

Systèmes d'Exploitation des Serveurs

Les systèmes d'exploitation des serveurs sont à 90% basés sur Linux. Contrairement aux interfaces graphiques des ordinateurs personnels, les serveurs n'ont généralement pas d'interface graphique. Les commandes sont exécutées via la ligne de commande, ce qui nécessite une bonne compréhension des commandes Linux.


      
      
Quelques commandes Linux de base :

      
ls - Liste les fichiers et dossiers
cd - Change de répertoire
mkdir - Crée un nouveau répertoire

Modèle OSI et Réseaux

Le modèle OSI (Open Systems Interconnection) est une norme qui définit les différentes couches de communication dans un réseau. Il est composé de 7 couches : physique, liaison de données, réseau, transport, session, présentation, et application. Pour relier plusieurs ordinateurs, on utilise un switch, créant ainsi un réseau local (LAN).

Astuce : Pour en savoir plus sur le modèle OSI, vous pouvez suivre le parcours d'apprentissage de Cisco Academy IT Essential.

Exercices Pratiques

  1. Installez un serveur Linux sur une machine virtuelle et essayez de créer un répertoire et de lister ses fichiers.
  2. Configurez un serveur web simple (comme Apache) sur votre serveur Linux et accédez-y via un navigateur web.
