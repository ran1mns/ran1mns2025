Titre du Cours : Introduction à l'Hébergement Web et aux Serveurs

Introduction

Ce cours vous introduit au monde de l'hébergement web et de l'administration de serveurs. Vous découvrirez les différentes manières d'héberger un site, le concept de virtualisation, les bases de la gestion d'un serveur Linux, et les bonnes pratiques de sécurité.

1. Les Types d'Hébergement Web

Choisir le bon hébergement est une étape clé pour votre projet.

Type Description Idéal pour Avantages Inconvénients
Mutualisé Votre site partage les ressources d'un serveur avec de nombreux autres. Blogs, sites vitrines, débutants. Peu coûteux, facile à gérer. Performances limitées, peu de contrôle.
VPS (Serveur Privé Virtuel) Une machine virtuelle avec des ressources dédiées sur un serveur physique partagé. Sites à trafic moyen, applications web. Bon rapport performance/prix, plus de contrôle. Nécessite des connaissances en administration.
Dédié Un serveur physique entièrement pour vous. Sites à fort trafic, applications critiques. Performances maximales, contrôle total. Coûteux, complexe à gérer.

2. La Virtualisation

La virtualisation permet de créer plusieurs environnements virtuels (machines virtuelles) sur une seule machine physique. C'est la technologie derrière les VPS. Elle est gérée par un logiciel appelé hyperviseur.

  • Type 1 (Bare Metal) : Installé directement sur le matériel (ex: VMware ESXi). Très performant, utilisé en production.
  • Type 2 (Hébergé) : Installé sur un système d'exploitation existant (ex: VirtualBox, VMware Workstation). Idéal pour les tests et le développement local.

3. Administration de base d'un serveur Linux

La plupart des serveurs web tournent sous Linux. La gestion se fait en ligne de commande.

Commandes Essentielles

  • ls : Lister les fichiers.
  • cd [dossier] : Changer de répertoire.
  • pwd : Afficher le répertoire actuel.
  • touch [fichier] : Créer un fichier vide.
  • mkdir [dossier] : Créer un répertoire.
  • rm [fichier] : Supprimer un fichier.
  • sudo apt update : Mettre à jour la liste des paquets (Debian/Ubuntu).

4. Sécurité et Connexion

Connexion via SSH (Secure Shell)

SSH est le protocole standard pour se connecter de manière sécurisée à un serveur distant. L'authentification par clés (une clé privée sur votre machine, une clé publique sur le serveur) est beaucoup plus sécurisée que l'authentification par mot de passe.


# Générer une paire de clés SSH sur votre machine locale
ssh-keygen -t rsa -b 4096

# Copier la clé publique sur le serveur
ssh-copy-id utilisateur@adresse_du_serveur

# Se connecter
ssh utilisateur@adresse_du_serveur

Sécurité des Mots de Passe

Si vous devez utiliser des mots de passe, assurez-vous qu'ils sont longs, complexes et uniques. Des gestionnaires de mots de passe comme KeePass ou Bitwarden sont fortement recommandés.

Exercices Pratiques

  1. Installer VirtualBox : Téléchargez et installez VirtualBox sur votre ordinateur.
  2. Créer une Machine Virtuelle : Créez une nouvelle VM et installez-y une distribution Linux légère comme Lubuntu.
  3. Pratiquer les commandes de base : Dans votre VM, utilisez le terminal pour créer une structure de dossiers : projets/site_web/images. Créez un fichier index.html dans le dossier site_web.
