DHCP et adresse IP

Le DHCP (Dynamic Host Configuration Protocol) alloue automatiquement une adresse IP aux équipements du réseau. Il s'agit d'un service (programme) qui tourne en permanence, souvent installé sur le routeur ou sur une machine (physique ou virtuelle).

Chaque adresse IP est unique sur le réseau. L'adresse IPv4 est composée de 4 nombres :

Exemple IPv4 : |__|.|__|.|__|.|__|

Remarque : Le dernier octet à 0 est réservé pour l'identifiant réseau dans certaines plages, il ne doit donc pas être utilisé pour un équipement.

Classes d'adresses IP

Les adresses IP étaient historiquement classées :

  • Classe A : 0 à 126 → numéro réseau sur la première position, numéro d'hôte sur les 3 suivantes
  • Classe B : 128 à 191 → numéro réseau sur les 2 premières positions, numéro d'hôte sur les 2 suivantes
  • Classe C : 192 à 223 → numéro réseau sur les 3 premières positions, numéro d'hôte sur la dernière
  • Classes D et E existent mais ne sont pas accessibles aux utilisateurs

La classe détermine la taille maximale du réseau :

  • Classe A : 255 * 255 * 255 * 126 IP
  • Classe B : 255 * 255 IP
  • Classe C : 255 IP

Le système de classes n’est plus utilisé sur Internet mais reste pertinent pour les réseaux locaux.

Plages privées : 10.x.x.x, 172.x.x.x, 192.x.x.x → réservées aux réseaux locaux, pas accessibles sur Internet.

IPv6

Pour gérer le grand nombre d’équipements, IPv6 a été créé :

Exemple IPv6 : |__|.|__|.|__|.|__|.|__|.|__|.|__|.|__|

Chaque bloc correspond à 16 bits (2 octets), représentés en hexadécimal, pour permettre un très grand nombre d’adresses uniques.

Le système hexadécimal utilise 16 symboles : 0-9 et A-F (majuscules ou minuscules).

Hébergement et DNS

Pour qu’un utilisateur accède à notre site :

  • On dépose notre dossier sur le serveur
  • Le routeur gère l'accès vers le serveur hébergeant le site
  • Le DNS (Domain Name System) fait le lien entre le nom de domaine et l'adresse IP
  • Le FAI interroge les serveurs DNS pour trouver le site demandé

Le DNS garantit que le nom de domaine correspond à la bonne adresse IP.

Noms de domaine

  • Hébergement mutualisé : plusieurs utilisateurs déposent leurs dossiers sur le même serveur
  • Nom de domaine = nom + extension (géographique : .fr, .bzh, .alsace ; thématique : .com, .edu, .gov, .org)
  • Sous-domaines possibles : sousdomaine.nom.extension
  • who.is : permet de connaître le propriétaire d’un nom de domaine

VPS, VM et cloud computing

  • VPS (Serveur Privé Virtuel) : souvent sous Linux (ex : Debian)
  • Location d’un serveur physique entier possible
  • Différence VM vs containers :
    • VM (Virtual Machine) : machine virtuelle complète
    • Container : compartiment léger pour exécuter des applications
  • Cloud computing : hébergement et déploiement sur des infrastructures partagées ou dédiées
